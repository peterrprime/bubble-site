GreenArc Biscayne

A self-sustaining community for Bird Key, Biscayne Bay, generated from a spreadsheet. Ten programs — residential, medical, agricultural, technical, and so on — each sized by its own space-program data, then packed by the bubble-adjacency method: overlap by adjacency, scale by square footage, maximize intersections, read circulation off the result. The Grasshopper/Python script below runs that method — it's the piece that turns the spreadsheet into a building.

South Beach Pavilion

Pedestrian flow through the block was simulated agent-by-agent — entries, alleyway use, combined movement — and the result was used to derive the pavilion's circulation and footprint directly, instead of guessing at it.

Micro Housing

A façade generated by a QuadTree algorithm: scatter points across the elevation, thin them out, subdivide the area recursively based on point density, then extrude each resulting cell to a different height.

This Site — Bubble Layout Editor

The bubble-adjacency method from GreenArc Biscayne — the same overlap/scale/circulation logic — rebuilt as a standalone tool: a metaball node graph where every bubble is draggable, resizable, and expandable, with the field re-solving live on a WebGL2 canvas. It runs the site you're on right now. No frameworks, no build step, one HTML file.

About

I design with code.

My thesis generated a building out of a spreadsheet — a space program pushed through an adjacency algorithm into a Grasshopper/Python script that produced the massing, the structure, and the circulation. Elsewhere: a façade grown from a QuadTree, and a building whose circulation was derived from a simulation of the people walking through it.

Then I took the bubble-adjacency method from that thesis and built it into a standalone web app, because doing it by hand was the slow part. That instinct — this is repetitive, so build the tool — is the thing I actually do.
